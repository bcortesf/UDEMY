¿No me reconoce los templates dentro de src/main/resources/templates?
WARN[0;39m [35m11076[0;39m [2m---[0;39m [2m[ restartedMain][0;39m [36mion$DefaultTemplateResolverConfiguration[0;39m [2m:[0;39m Cannot find template location: classpath:/templates/ (please add some templates, check your Thymeleaf configuration, or set spring.thymeleaf.check-template-location=false)

Hola! Comparto la solución que encontré y me funcionó: click derecho en el proyecto > run as > run configurations... > click en la pestaña Classpath > click en User Entries > click en Advanced... > seleccionar Add Folders y darle OK > y finalmente agregar la carpeta "resources". Das click en ok, aplicar, run. Y levanta el proyecto y actualizar el navegador.

Profesor un saludo, tengo una duda, si realmente las dos formas que tiene Spring de implementar la redireccion  pertenecen a la Api de los servlet, es decir si es lo mismo hacer un response.sendRedirect("...") o un request.getRequestDispatcher().forward("....") y de ser asi como Spring hace que a través de solo "forward:/ruta" o "redirect:/ruta" realmente funcione


Hola Alexander, exactamente, así es, spring lo hace con forward para el Dispatcher y redirect para sendRedirect.
La diferencia es que forward redirige a otro recurso o ruta pero dentro del mismo request, sin recargar la página, sin iniciar una nueva petición http, mientras que redirect redirige a una página o ruta iniciando una nueva petición http, por lo tanto todos los datos que teníamos en el request como parámetros y atributos se pierden ya que se reinicia la petición.

No olvidar agregar la dependencia spring-boot-starter-validation en el pom.xml
Para validar es muy importante agregar la siguiente dependencia en el pom.xml del proyecto Spring Boot:

		<dependency>
			<groupId>org.springframework.boot</groupId>
			<artifactId>spring-boot-starter-validation</artifactId>
		</dependency>
Muy importante, es una actualización desde la versión de Spring Boot 2.3.0 o superior, agregar esta dependencia para validar!
